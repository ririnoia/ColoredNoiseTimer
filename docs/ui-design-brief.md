# UI Design Brief

## 1. デザイン方針

ColoredNoiseTimer は、とにかくシンプルな集中用ツールとして設計します。

## 2. UI言語

- UI文言: 日本語
- ノイズ名: 英語
- ページタイトル中の状態名: `Focus` / `Break`

## 3. デザインキーワード

- シンプル
- 静か
- 集中
- 余白がある
- 操作が少ない
- スマホで押しやすい

## 4. 画面構成

1ページ構成。

上から順に以下を配置します。

1. アプリ名
2. 現在の状態
3. タイマー表示
4. タイマー操作
5. ノイズ選択
6. 音声操作
7. 音量スライダー
8. 詳細設定
9. 将来広告枠

## 5. 推奨レイアウト

### スマホ

- 縦1カラム
- タイマーを大きく表示
- 主要ボタンは親指で押しやすいサイズ
- ノイズ選択はカードまたはセグメントボタン
- 詳細設定は折りたたみでもよい

### PC

- 中央寄せの1カラム
- 最大幅は狭めにする
- 情報量を増やしすぎない
- 広告枠はページ下部

## 6. 主要コンポーネント

- `TimerDisplay`
- `TimerControls`
- `ModeToggle`
- `NoiseSelector`
- `AudioControls`
- `VolumeSlider`
- `TimerSettings`
- `EndSoundToggle`
- `FutureAdSlot`

## 7. Figma MCP利用方針

Figma MCP / Claude Code公式Figmaプラグインを使ってUI設計を行います。

Claude Codeに依頼する際は、以下を明示します。

- 日本語UI
- ノイズ名だけ英語
- スマホファースト
- 1ページ構成
- とにかくシンプル
- Tailwind CSSで実装しやすい余白・サイズにする
- 装飾よりも操作しやすさを優先する

## 8. UI文言案

- アプリ名: ColoredNoiseTimer
- 状態: 集中 / 休憩
- ボタン:
  - タイマー開始
  - タイマー停止
  - リセット
  - ノイズ再生
  - ノイズ停止
- 設定:
  - 集中時間
  - 休憩時間
  - 終了音
  - 音量
- 将来広告枠:
  - MVPでは非表示
